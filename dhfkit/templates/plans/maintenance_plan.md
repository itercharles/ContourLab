# Software Maintenance Plan

## Purpose
This document describes the software maintenance process in accordance with
IEC 62304 §6.

## Feedback Monitoring
Post-production feedback is monitored through:
- User-reported issues tracked as CR items in the DHF
- Post-market surveillance inputs reviewed at regular intervals

## Problem Reporting
Each problem detected in released software is documented as a CR item,
including a description of the problem and its criticality assessment.

## Problem Evaluation
Each CR is evaluated to determine:
- Whether a software change is required
- The effect on safety of released software
- Required risk management activities

## Change Implementation
Changes to released software follow the software development process (Clause 5)
as appropriate to the scope of the change.

## Re-release
Modified software is re-released in accordance with §5.8, including version
documentation and delivery verification.
